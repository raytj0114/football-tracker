実装した変更を検証する。**実装完了前に必ず実行すること。**

## 検証手順

以下を順番に実行し、すべてパスするまで修正を繰り返す。

### Step 1: 静的解析

```bash
npm run type-check
npm run lint
```

エラーがあれば修正し、再度実行。

### Step 2: ビルド確認

```bash
npm run build
```

ビルドエラーがあれば修正し、Step 1から再実行。

### Step 3: テスト

```bash
npm run test
```

失敗するテストがあれば、実装またはテストを修正。

### Step 4: 動作確認

```bash
npm run dev
```

- http://localhost:3000 にアクセス
- 変更した機能を実際に操作
- ブラウザのコンソールにエラーがないことを確認
- 既存機能が壊れていないことを確認

## 検証結果の報告

各ステップの結果を以下の形式で報告する：

- [ ] 型チェック: パス / エラーN件
- [ ] リント: パス / エラーN件
- [ ] ビルド: 成功 / 失敗
- [ ] テスト: パス / 失敗N件
- [ ] 動作確認: 正常 / 問題あり

問題があった場合は、原因と修正内容も報告する。
